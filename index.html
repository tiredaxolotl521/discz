Background

For our Iterative Design project, we choose Discz from Y combinator. The core conecpt behind Discz is that it is a Spotify-integrated music discovery platform built to help you find your next favorite artists and songs. It's main functionality is the ability to build playlists at the swipe of a finger and find new music via custom recommendations catered to your taste! You can swipe through song snippets to find music you love and add it to your spotify playlists.

Critiques

Pros

  • I like the overall layout and heirarchy.
  • The navigation bar works well and shows which page you are on.
  • I like the transition between pages.
  • I like that you can see the album behind it, which shows that you should swipe in some direction to see the next album.

Cons

  • The text is kind of small (describing the genres), but the big pictures are good.
  • Discover and the playlists screens are too similar.
  • Not sure what the "Create" button is supposed to do. It should go to a Create page.
  • Recommendations and recent listening are within the settings option of the app, which is a bit unintuitive. I thought the recommendations were all the songs you haven't listened to.
  • The plus and minus buttons in the settings under My Recommendations are confusing.
  • I was confused about the "Skipping" function in the Discover page. Does skipping count as liking or disliking?
  • I'm confused about the cancel button in the player screen (the X).

Scenario

Imagine you are a music listener and want to discover more songs similar to your taste. This app allows you to explore and discover new music with a tinder-style swiping action. This is not an actual app but an interactive mockup made on prototyping software.

Testing Instructions

  1. Select a radio in the Discover page and swipe through songs. The only radio that is currently interactable is the K-pop radio.
  2. Create a new radio and add a minimum of two songs.
  3. View the first playlist on the Library page.
  4. View your profile and view the edit profile page.

Post-Test Questions

  1. How intuitive was the "Library" page?
  2. How intuitive was the "Discover" page?
  3. How intuitive was the "Profile" page?
  4. Please share any overall thoughts or improvements you would like to see on this app. Any and all thoughts are welcome!

Testing Summary

  • Testers able to use the swipping action after some trial and error
  • Testers challenged by the "Create Radio" task, unclear where the button to create radio is located. Need to clarify this task to make it less challenging.
  • Testers knew to add two songs to the first two highlighted discs on screen.
  • One tester noted being able to see more songs in the Recommended & Recently Listened sections on the Profile page.
  • Testers noticed similarity to Tinder
  • Overall gave 4.5 average intutive scores for Library, Discover, and Profile page.
  • Clear to use buttons, not information overload for the user

Potential Changes

  • Standardize the terminology for creating a Radio on our Discover Page
  • Make the swipping action more seamless as user was unsure how to swipe at first
  • Have the ability to edit and change the Recommendations and Recently Listened on the Profile Page
  • Consider adding a thumbs up button similar to Pandora.
  • Allow users to add three mandatory songs, not only two songs.
  • Add the word "My" to Library page (ie. "My Library")
